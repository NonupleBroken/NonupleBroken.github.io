选择明文攻击

适用情况：

对输入的任意明文服务器返回 RSA 加密结果。

可以通过选择明文攻击来获取 nnn。

原理

首先发送 222，让服务器进行加密，返回 c2=2e mod nc_2 = 2^e \bmod nc2​=2emodn

继续发送 222^222，让服务器进行加密，返回 c4=4e mod nc_4 = 4^e \bmod nc4​=4emodn

不妨设 2e=a+bn2^e = a + bn2e=a+bn，有：

c2=(a+bn) mod n=ac4=(a2+2abn+b2n2) mod n=a2 mod n\begin{aligned} c_2 & = (a + bn) \bmod n = a \\ c_4 & = (a^2 + 2abn + b^2n^2) \bmod n = a^2 \bmod n \end{aligned} c2​c4​​=(a+bn)modn=a=(a2+2abn+b2n2)modn=a2modn​

所以 c22c_2^2c22​ 和 c4c_4c4​ 模 nnn 同余

即：

c22−c4=knc_2^2 - c_4 = kn c22​−c4​=kn

同理：

c23−c8=tnc_2^3 - c_8 = tn c23​−c8​=tn

一般来说 a2a^2a2 比 nnn 大，所以 kkk 不为 000。

同理还可以构造更多的例子取他们的公因数，来更加确定地找 nnn。

选择密文攻击

适用情况：

可以构造任意密文并获得对应明文。

通过选择密文攻击获得特定的明文。

原理

假设服务器创建了密文 C=Me mod nC = M^e \bmod nC=Memodn，并且把 ccc 发送给用户，用户可以发送任意密文服务器返回解密后的明文。可以通过下面方法求出 MMM：

  1. 选择任意的 XXX 与 nnn 互素。
  2. 计算 Y=CXe mod nY = CX^e \bmod nY=CXemodn。
  3. 由于可以进行选择密文攻击，那么可以求得 YYY 对应的解密结果 Z=YdZ = Y^dZ=Yd。
  4. 则 Z=Yd=(CXe)d=CdX=MX mod nZ = Y^d = (CX^e)^d = C^dX = MX \bmod nZ=Yd=(CXe)d=CdX=MXmodn。由于 XXX 与 nnn 互素，很容易求得相应的逆元，进而可以得到 MMM。

RSA Parity Oracle

适用情况：

可以选择密文并泄露最低位。

服务器会对一个给定的密文进行解密，并且会检查解密的明文的奇偶性，并根据奇偶性返回相应的值，比如 111 表示奇数，000 表示偶数。那么给定一个加密后的密文，我们只需要 log2nlog_2 nlog2​n 次就可以知道这个密文对应的明文消息。

原理

假设：

C=Me mod nC = M^e \bmod n C=Memodn

第一次时，构造密文：

C×2e mod n=(2M)e mod nC \times 2^e \bmod n = (2M)^e \bmod n C×2emodn=(2M)emodn

服务器解密后得到：

2M mod n2M \bmod n 2Mmodn

这里：

  • 2M2M2M 是偶数，它的幂次也是偶数。
  • nnn 是奇数，因为它是由两个大素数相乘得到。

那么：

  • 服务器返回奇数，即 2M mod n2M \bmod n2Mmodn 为奇数，则说明 2M>n2M > n2M>n，且减去了奇数个 nnn，又因为 2M<2n2M < 2n2M<2n，因此减去了一个 nnn，即 n2≤M<n\frac{n}{2} ≤ M < n2n​≤M<n。
  • 服务器返回偶数，则 2M<n2M < n2M<n，即 0≤M<n20 \leq M < \frac{n}{2}0≤M<2n​。

第二次时，构造密文 C×4e mod nC \times 4^e \bmod nC×4emodn，服务器解密后得到 4M mod n4M \bmod n4Mmodn，判断其奇偶性可以知道 mmm 和 n4\frac{n}{4}4n​ 的大小关系。

以此类推，第 iii 次时，构造密文 C×2ie mod nC \times 2^{ie} \bmod nC×2iemodn，服务器解密后得到 2iM mod n2^i M \bmod n2iMmodn，判断其奇偶性可以知道 mmm 和 n2i\frac{n}{2^i}2in​ 的大小关系。

所以我们就有了一个二分算法，可以在 log2nlog_2 nlog2​n 次内将 mmm 的范围逼近到一个足够狭窄的空间。

RSA Byte Oracle

适用情况：

可以选择密文并泄露最后一个字节。

服务器会对一个给定的密文进行解密，并且会给出明文的最后一个字节。那么给定一个加密后的密文，我们只需要 log256nlog_{256}nlog256​n 次就可以知道这个密文对应的明文消息。

原理

这是 RSA Parity Oracle 的扩展，既然可以泄露出最后一个字节，那么按道理获取密文对应明文的次数应该可以减少。

假设：

C=Me mod nC = M^e \bmod n C=Memodn

第一次时，构造密文：

C×256e=(256M)e mod nC \times 256^e = (256M)^e \bmod n C×256e=(256M)emodn

服务器解密后得到：

256M mod n256M \bmod n 256Mmodn

这里：

  • 256M256M256M 是偶数。
  • nnn 是奇数，因为它是由两个大素数相乘得到。

由于 M=Cd mod nM = C^d \bmod nM=Cdmodn，所以 MMM 小于 nnn，那么：

256M mod n=256M−kn,(k<256)256M \bmod n = 256M − kn, \quad (k < 256) 256Mmodn=256M−kn,(k<256)

而且对于两个不同的 k1k_1k1​、k2k_2k2​，有：

256M−k1n mod 256≠256M−k2n mod 256256M − k_1 n \bmod 256 \neq 256M − k_2 n \bmod 256 256M−k1​nmod256​=256M−k2​nmod256

同时 256M−kn256M − kn256M−kn 的最后一个字节其实就是 −kn−kn−kn 在模 256256256 的情况下获取的。

那么，我们可以首先枚举出 0−2550 - 2550−255 情况下的最后一个字节，构造一个 kkk 和最后一个字节的映射表 map。

当服务器返回最后一个字节，那么我们可以根据上述构造的映射表得知 kkk，即减去了 kkk 个 nnn，即 kn≤256M≤(k+1)nkn \leq 256M \leq (k+1)nkn≤256M≤(k+1)n。

以此类推，第 iii 次时，构造密文：

C×256ieC \times 256^{ie} C×256ie

服务器解密后得到：

256i×M mod n=256i×M−kn256^i \times M \bmod n = 256^i \times M − kn 256i×Mmodn=256i×M−kn

即：

kn≤256i×M<n+knkn256i≤M<(k+1)n256ikn \leq 256^i \times M < n+kn \\ \frac{kn}{256^i} \leq M < \frac{(k+1)n}{256^i} kn≤256i×M<n+kn256ikn​≤M<256i(k+1)n​

那么，可以设初始范围为：

Mn∈[L0,R0],(L0=0,R0=1)\frac{M}{n} \in [L_0,R_0], \quad (L_0 = 0, R_0 = 1) nM​∈[L0​,R0​],(L0​=0,R0​=1)

第i次迭代后：

Li=Li−1+ki256iRi=Li−1+ki+1256iL_i = L_{i-1} + \frac{k_i}{256^i} \\ R_i = L_{i-1} + \frac{k_i + 1}{256^i} Li​=Li−1​+256iki​​Ri​=Li−1​+256iki​+1​

还有最后一个问题，就是迭代的次数。由于明文的最后一个字节可以直接由服务器解密得到，那么只需要限定 MMM 的范围至：

Mmax−Mmin<256M_{max} - M_{min} < 256 Mmax​−Mmin​<256

即：

n×(Ri−Li)=n256i<256n \times (R_i - L_i) = \frac{n}{256^i} < 256 n×(Ri​−Li​)=256in​<256

由于 nnn 是由 512 bit 的 ppp 和 qqq 相乘得到，所以 n<21024n < 2^{1024}n<21024，那么由上式可得：i≥128i \geq 128i≥128。

一般对于这样的 Oracle，最多需要 log⁡2bitsn\log_{2^{bits}} nlog2bits​n 次迭代即可确定明文，其中 bits 为泄露的明文 bit 数。RSA Parity Oracle 为 1，RSA Byte Oracle 为 8。
